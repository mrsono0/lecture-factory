# IT 기술 강의 구성안 설계 — 에이전트 팀 & 워크플로우

> **최종 산출물: 강의 구성안(커리큘럼/세션 구조/시간 배분/학습목표 정렬표)**
> 교안 본문(강사용 대본, 핸드아웃, 실습 가이드 등)은 이 워크플로우의 범위 밖입니다. **단, 교안 작성을 위한 “프롬프트 향상 패키지(생성/QA/개선 루프)”는 구성안에 포함**합니다.
>
> **단일 파일 완결본 원칙(필수)**: 최종 산출물은 `최종_강의구성안.md` **1개 파일만으로도** 강의 구성안 전체를 이해/운영할 수 있도록 **모든 핵심 내용(표/세션 상세/LO/시간/평가/환경/차별화/QA/교안 프롬프트 패키지)을 본문에 포함**합니다. (개별 단계 산출물 파일은 추적/버전 관리용으로 유지)

## 1) 에이전트 팀 구성(권장 8인)

| ID | 에이전트 | 핵심 역할(강의 구성안 관점) | 주요 산출물 |
|---|---|---|---|
| 0 | **오케스트레이터** | 전체 일정/의존성/승인 게이트 운영, 충돌 해결 | 작업지시서, 버전 관리, 최종 승인 |
| 1 | **트렌드 리서처** | 커리큘럼이 시장·표준과 어긋나지 않게 근거 제공 | 트렌드/표준 브리프, 권장 스택/버전 |
| 1B | **도메인 SME** | 기술 정확성, 현업 관행, 보안/운영 관점 타당성 검증 | 사실 검증 리포트, 커리큘럼 타당성 의견 |
| 2 | **학습설계 아키텍트** | 학습목표-활동-평가 정렬, 난이도 곡선 설계 | LO 맵, 세션 목표, 평가 매트릭스 |
| 3 | **콘텐츠 스트럭처러** | 장/절/세션 흐름 설계, 시간 배분 | 상세 목차, 세션별 구성표, 타임라인 |
| 5B | **학습자 분석 전문가** | 페르소나/선수지식/탈락 지점 예측, 브리지 설계 | 페르소나, 선수지식 체크, FAQs/오해 포인트 |
| 5A | **QA 검증 에이전트** | 누락/중복/시간 과부하/정렬성/용어 일관성 점검 | QA 결함 리스트(P0~P2), 수정요청 |

> **소규모 운영(최소 4인)**: 0 오케스트레이터 + 1B 도메인 SME + 2 학습설계 아키텍트 + 3 콘텐츠 스트럭처러

## 2) 강의 구성안 설계 워크플로우

| 단계 | 주관 | 순서 | 입력 | 출력 | 품질 게이트 |
|---|---|---|---|---|---|
| 0. 스코프 고정 | 0 + 5B | 순차 | 강의 주제/대상/시간/제약 | 범위 정의, 톤, 산출물 목록, **교안 프롬프트 운영 규격(출력 포맷/언어/분량/품질 기준)** | **범위 크리프 방지**, 대상/전제 명시, 실습 환경/비용/계정 제약 확정 |
| 1. 트렌드·현업 근거 확보 | 1 + 1B | 병렬 | 주제/스택 후보 | 표준/버전/현업 관행 브리프 | **근거 출처 확보**, 비현업 내용 제거 |
| 2. 학습자 분석 | 5B | 순차 | 대상 정보 | 페르소나, 선수지식 갭, 예상 난점 | **대상별 맞춤 설계 근거** 확보 |
| 3. 학습목표(LO) 설계 | 2 + 5B | 병렬→합류 | 브리프, 학습자 분석 | LO 목록, 난이도 곡선, 평가 정렬 | **LO-활동-평가 정렬** 완료 |
| 4. 커리큘럼 구조 설계 | 3 + 2 + 1B | 순차 | LO, 기술 범위 | 장/절/세션 구조, 시간 배분 | **흐름 점진성**, 용어·개념 선후관계 |
| 5. 세션별 상세 구성 | 3 + 2 | 순차 | 커리큘럼 구조 | 세션별 목표·주제·활동 유형·시간 배분표 | **세션 단위 완결성**, 시간 초과 없음 |
| 5.5 통합본(드래프트) 생성 | 0 | 순차 | 00~05 산출물 | `최종_강의구성안.md` 드래프트(**단일 파일 완결본 + 교안 프롬프트 패키지 포함**) | 통합본 최신화, **단일 파일 자급자족(외부 파일 참조 없이 핵심 내용 포함)**, 표/링크 깨짐 0, 총 시간 합 1차 검증, **프롬프트 향상 사이클 실행 가능(최종 파일만으로)** |
| 6. QA 게이트 | 5A | 순차 | `최종_강의구성안.md` 드래프트 | 결함(P0~P2), 반려/승인 | **P0=0**, 누락·중복·시간 불일치 0, **단일 파일 완결본 + 교안 프롬프트 패키지 요건 충족** |
| 7. 차별화 폴리싱 | 7 + 0 | 병렬 | 승인된 구성안 | 차별화 포인트 반영, 최종 승인 | **차별점 3~5개** 확보 |

### 운영 규칙(합류/결정/QA)

#### 병렬→합류 기준(Definition of Ready)

- **1단계 합류 조건**: 출처 링크 포함, 권장 스택/버전 명시, “왜 이 버전/스택인가” 근거 1~2줄, 리스크/대안(옵션) 표기, 1B 사실 검증 완료
- **3단계 합류 조건**: 모든 LO가 Bloom 동사로 작성, 각 LO가 *활동*과 *평가*에 최소 1개 이상 매핑, 선수지식/브리지 필요 시 표시, 5B 검토 완료

#### 의사결정/충돌 해결(우선순위)

- 사실/보안/운영 타당성: **1B 우선**
- LO-활동-평가 정렬: **2 우선**
- 구조/흐름/시간 배분: **3 우선**
- 최종 트레이드오프 결정 및 기록: **0 최종** (결정 로그는 `00_범위정의/범위정의서.md`에 남김)

#### QA 결함 등급(권장)

- **P0**: 즉시 수정(예: 시간 합 불일치, LO 누락/중복, 세션 단위 미완결, 용어/개념 선후관계 붕괴, **교안 프롬프트 패키지 누락/불완전(최종 파일만으로 향상 사이클 불가)**)
- **P1**: 수정 권장(예: 난이도 곡선 급변, 활동 유형 편중, 평가-LO 매핑 약함)
- **P2**: 폴리싱(예: 표현/용어 통일, 표 가독성, 차별화 문구 정리)

## 3) 최종 산출물: 강의 구성안 표준 규격

### 강의 구성안에 포함되어야 하는 항목

| 항목 | 설명 |
|---|---|
| **강의 개요** | 강의 제목, 대상, 총 시간, 회차 수, 목적/배경 |
| **학습목표(LO) 목록** | 수강 후 달성해야 할 역량 (Bloom 기반 동사 사용) |
| **대상 분석** | 수강생 페르소나, 선수지식 요구사항, 예상 난점 |
| **커리큘럼 구조** | 전체 회차별 대주제, 세부 주제 목록, 흐름도 |
| **세션별 상세표** | 각 세션의 LO, 핵심 주제, 활동 유형(이론/데모/실습/토론), 시간 배분(분) |
| **난이도 곡선** | 회차별 난이도 흐름 (입문 → 핵심 → 심화 → 통합) |
| **평가 전략** | 각 LO에 대응하는 평가 방법 (퀴즈/과제/프로젝트) 개요 |
| **실습 환경 요약** | 필요 스택, OS, 계정, 비용 등 환경 제약 요약 |
| **차별화 포인트** | 경쟁 강의 대비 차별 요소 3~5개 |
| **교안 프롬프트 패키지(필수)** | `최종_강의구성안.md` **단독으로** 교안 생성/QA/개선(프롬프트 향상) 사이클을 돌릴 수 있는 템플릿/루브릭/세션별 입력 스펙 |

### 단일 파일 완결본(필수) 체크리스트

- `최종_강의구성안.md` **단일 파일에 아래 항목이 모두 포함**되어야 한다(링크로 “참조”만 하고 본문이 비어 있으면 실패).
  - 강의 개요(대상/시간/환경/목적)
  - 대상 분석(페르소나/선수지식/난점/브리지)
  - LO 목록 + LO-활동-평가 정렬표 + 난이도 곡선
  - 커리큘럼 구조(흐름도 포함 권장)
  - **세션별 상세표(모든 세션)**: LO, 핵심 주제, 활동 유형, 시간(분), 산출물
  - 평가 전략(산출물/퀴즈/발표 등)
  - 실습 환경 요약(제약/대안)
  - 차별화 포인트(3~5개) + USP 문구(선택)
  - QA 요약(P0/P1/P2, 시간 합/정렬성 체크)
  - **교안 프롬프트 패키지(필수)**: 아래 4개가 “최종 파일 안에” 모두 있어야 함  
    - 교안 출력 규격: 산출물 형태(대본/슬라이드/핸드아웃/실습가이드), 포맷(Markdown 등), 언어, 분량, 톤, 금지/허용 규칙  
    - 세션별 PromptSpec: 모든 세션에 대해 **목표/LO/시간/활동/산출물/비유/오해포인트/검증 기준**이 표 형태로 정리  
    - 프롬프트 향상 사이클 Runbook: Generate → QA(루브릭) → Revise → Finalize 절차 + “중단 조건/합격 기준”  
    - QA 루브릭: LO 정렬/시간·실습비율/정확성/학습자 난점 대응/용어 일관성/산출물 충족 체크리스트  
- 권장 부록(선택): 미니프로젝트 요구사항 요약, 운영 체크리스트

## 4) 시작에 필요한 입력(6개)

1) **강의 주제/스택** (예: Kubernetes 운영, Spring AI, 데이터 파이프라인 등)
2) **대상 수준/배경** (초급/전환/경력)
3) **총 시간/회차** (예: 8회, 회당 2시간)
4) **강의 구성안 산출 범위** (커리큘럼만 / 세션 상세표 포함 / 평가 전략 포함)
5) **실습 환경 제약** (로컬/클라우드, OS, 계정, 비용 상한)
6) 수강 후 **반드시 할 수 있어야 하는 것 3가지**

### 추가 입력(프롬프트 패키지용, 미입력 시 기본값 적용)

7) **교안 산출물 형태/포맷** (예: 강사용 대본+실습 가이드+슬라이드 개요, Markdown)  
8) **교안 톤/스타일 규칙** (예: “AI 시대의 서사” 비유 톤 강도, 금지어/표현)  
9) **프롬프트 향상 목표/합격 기준** (예: 실습 60%+, 세션당 퀴즈 포함, 오류·오해 포인트 포함)

## 5) 산출물 폴더 구조 및 파일 정리 규칙

### 기본 원칙

- 워크플로우 실행 시 **가장 먼저** 산출물 폴더를 생성한다.
- 폴더명 형식: **`YYYY-MM-DD_강의제목`** (예: `2026-02-14_Kubernetes_운영`)
- 모든 최종 산출물과 참고 자료는 이 폴더 안에서 작성·관리한다.

### 폴더 구조

```
YYYY-MM-DD_강의제목/
├── 00_범위정의/              # 단계 0 — 스코프 고정
│   └── 범위정의서.md
├── 01_트렌드_근거/           # 단계 1 — 트렌드·현업 근거
│   ├── 트렌드_브리프.md
│   └── 참고자료/             # 출처 원문, 링크 모음 등
├── 02_학습자분석/            # 단계 2 — 학습자 분석
│   └── 페르소나_분석.md
├── 03_학습목표/              # 단계 3 — LO 설계
│   └── LO_맵.md
├── 04_커리큘럼구조/          # 단계 4 — 커리큘럼 구조 설계
│   └── 커리큘럼_구조표.md
├── 05_세션상세/              # 단계 5 — 세션별 상세 구성
│   └── 세션별_상세표.md
├── 06_QA/                    # 단계 6 — QA 게이트
│   └── QA_결함리스트.md
├── 07_차별화/                # 단계 7 — 차별화 폴리싱
│   └── 차별화_포인트.md
├── 08_프롬프트패키지/         # (선택) 교안 프롬프트 패키지(추적용)
│   └── 교안_프롬프트_패키지.md
├── 최종_강의구성안.md         # ★ 최종 통합 산출물(단일 파일 완결본)
└── 참고자료/                  # 전체 공용 참고 자료
    ├── 경쟁강의_분석.md
    └── 기타_레퍼런스/
```

### 파일 정리 규칙

| 규칙 | 설명 |
|---|---|
| **폴더 자동 생성** | 오케스트레이터(0)가 워크플로우 시작 시 위 폴더 구조를 자동 생성 |
| **파일명 규칙** | 한글 허용, 공백 대신 `_` 사용, 확장자는 `.md` 기본 |
| **버전 관리** | 주요 변경 시 파일명 끝에 `_v2`, `_v3` 접미사 추가 (예: `LO_맵_v2.md`) |

## 1. AI 시대의 서사 (Narrative of the AI Era)

### 정의
AI가 코드를 대신 작성해 주는 시대에, 개발자의 핵심 경쟁력이 '코드를 타이핑하는 사람(How)'에서 '무엇을 만들지 문제를 정의하는 사람(What)'으로 완전히 변화했다는 패러다임의 전환을 의미합니다[1][2].

### 핵심 개념
- **문제 정의 vs 문제 해결**: AI는 '어떻게 만들 것인가(문제 해결)'에는 탁월하지만, 사용자가 진짜 원하는 것이 무엇인지 '무엇을 만들 것인가(문제 정의)'를 결정하는 것은 여전히 인간의 고유 영역[3]
- **말이 되어야 프로그램이 된다**: 코딩 문법을 암기하는 것보다, 달성하고자 하는 목적과 조건을 논리적이고 명확한 명세서로 표현하는 능력이 가장 중요[4][5]

### 실습 예시
"AI에게 단순히 '로그인 기능 만들어줘'라고 지시하기 전에, 어떤 정보(이메일/아이디)로 로그인할지, 실패 시 어떻게 할지 등 '무엇'이 필요한지 직접 기획해 보기"[1][6]

## 2. Antigravity 설치 (Antigravity Installation)

### 정의
구글이 개발한 AI 에이전트 중심의 최신 IDE(통합 개발 환경)로, 단순한 코드 자동 완성을 넘어 에이전트가 주도적으로 계획하고 터미널과 브라우저를 제어하며 개발을 돕는 플랫폼[7]

### 핵심 개념
- **에이전트 관리자 (Agent Manager)**: 여러 에이전트에게 동시다발적으로 작업을 지시하고 진행 상황을 이메일 받은편지함(Inbox)처럼 추적·관리하는 미션 컨트롤 대시보드[10]
- **검토 기반 개발 (Review-driven development)**: AI가 마음대로 코드를 변경하거나 명령어를 실행하기 전 사용자에게 승인(Approve)을 받도록 하여 안전성을 보장하는 정책[13][14]

### 실습 예시
"Antigravity 공식 사이트에서 다운로드 후 설치를 진행하고, 초기 설정 시 '검토 기반 개발' 모드를 선택한 뒤 개인 Gmail 계정으로 로그인하여 첫 워크스페이스를 열기"[13]

## 3. Python 환경 구축 (Python Environment Setup)

### 정의
파이썬 프로그램을 실행하고 개발하기 위해 컴퓨터 시스템에 파이썬 인터프리터를 설치하고 세팅하는 기본 준비 과정[17][18]

### 핵심 개념
- **환경 확인**: 내 컴퓨터에 이미 파이썬이 있는지 터미널 명령어로 확인하는 기초 IT 리터러시[17][18]
- **AI와의 협업**: 강사나 구글 검색에 의존하지 않고, Antigravity 에이전트와 대화하며 환경을 파악하고 세팅을 안내받음[17]

### 실습 예시
"터미널을 열고 python --version 또는 python3 --version을 입력하여 설치 여부를 확인한 후, 없다면 Antigravity 에이전트에게 '내 OS에 맞는 파이썬 설치 명령과 절차를 알려줘'라고 프롬프트를 입력하여 지시대로 설치하기"[17][18]

## 4. uv 패키지 매니저 (uv Package Manager)

### 정의
기존의 pip 패키지 매니저를 대체하는 최신 도구로, 파이썬 라이브러리 패키지 다운로드 및 가상환경 생성 속도를 극대화해 주는 프로그램[19][20]

### 핵심 개념
- **스크립트 기반 설치**: 브라우저 다운로드 없이 터미널 명령어 한 줄(curl 또는 powershell)로 시스템에 설치[19]
- **파이썬 버전 관리**: 시스템에 여러 버전의 파이썬을 설치하고 프로젝트별로 특정 버전을 고정(pin)할 수 있도록 지원[23]

### 실습 예시
"OS에 맞춰 터미널에 설치 스크립트(curl -LsSf https://astral.sh/uv/install.sh | sh 등)를 실행하고 터미널 재시작 후 uv --version 명령어로 정상 설치되었는지 확인하기"[19]

## 5. 가상환경 (Virtual Environment)

### 정의
진행 중인 프로젝트마다 필요한 파이썬 버전과 라이브러리(패키지)들이 섞이거나 충돌하지 않도록 독립적으로 격리해 놓은 작업 공간[17]

### 핵심 개념
- **격리성**: A 프로젝트와 B 프로젝트가 요구하는 패키지 버전이 다를 때 발생하는 문제를 원천 차단[17]
- **도구 활용**: 파이썬 기본 기능 대신, 앞서 설치한 가장 빠른 도구인 uv 명령어를 활용해 가상환경을 생성하고 활성화[17]

### 실습 예시
"Antigravity 에이전트에게 '현재 폴더에 uv를 사용하여 파이썬 가상환경을 생성하고 활성화하는 명령어 절차를 알려줘'라고 지시하고, 응답받은 명령어를 터미널에 입력하여 경로 앞에 가상환경 표식(예: (.venv))이 뜨는지 검증하기"[17]

## 6. 첫 파이썬 코드 생성 (Generating First Python Code)

### 정의
문법을 일일이 타이핑하지 않고, AI 코딩 도구(codex 등)에 명확한 요구사항(프롬프트)을 전달하여 의도한 코드를 출력받는 과정[17][24]

### 핵심 개념
- **프롬프트 5대 체크리스트**: 코드를 요청할 때는 항상 1) 기능, 2) 입력, 3) 출력, 4) 제약 사항, 5) 예외 처리가 명시되어야 환각(엉뚱한 답변)을 막을 수 있음[25][26]
- **코드 리뷰**: AI가 만든 코드를 맹신하지 않고, 요구사항이 전부 반영되었는지, 경계값(0이나 100 등)과 예외를 잘 처리하는지 검증[26][27]

### 실습 예시
"에이전트에게 '나이 계산기를 만들어줘'라고 대충 말하는 대신, '입력: 출생연도, 출력: 현재 나이, 예외: 미래 연도 입력 시 에러 출력'처럼 구체적인 프롬프트를 작성하여 코드를 생성받고, 내 의도와 맞는지 확인하기"[25][27]

## 7. 예측-검증-설명 패턴 (Predict-Verify-Explain Pattern)

### 정의
코드를 무작정 실행(Run) 버튼부터 누르는 표면적 학습을 방지하기 위해, 결과를 먼저 '예측'하고, 실행하여 '검증'한 뒤, 왜 그런 결과가 나왔는지 '설명'하는 능동적 학습 방법론[28]

### 핵심 개념
- **깊은 이해(Deep Learning)**: 단순히 코드를 복사해서 붙여넣는 것을 넘어, 코드의 흐름과 컴퓨터의 동작 원리를 파악[28][29]
- **디버깅 근육**: 예측과 다른 결과(에러)가 발생했을 때 AI에게 원인을 집요하게 물어보는 습관을 통해 문제를 추적하는 역량을 기름[29]

### 실습 예시
"변수에 x = 10, x = 20, print(x)가 적힌 코드를 눈으로 보고 먼저 '무엇이 출력될까?' 예측한 뒤, 실행해서 20이 나오는 것을 검증하고 '변수 상자에는 마지막으로 넣은 값만 남기 때문'이라고 스스로 원리를 설명하기"[30]

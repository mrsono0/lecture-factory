### 1. 세션-095: 기존_클래스_확장하기_VIPCustomer와_super

**파일**: `02_Material/sessions/세션-095-기존_클래스_확장하기_VIPCustomer와_super_v1.0.md`
**위치**: 라인 51, 코드 블록 #1
**문제**: 함수 매개변수 연속 라인의 들여쓰기 오류
**상세**: 라인 3 (def __init__ 매개변수 계속)에서 17개 공백 사용 → PEP 8 위반 (4의 배수 아님)

**원본 코드**:
```python
class VIPCustomer(Customer):
    def __init__(self, name: str, email: str, age: int,
                 discount_rate: float = 0.1):  # ❌ 17 spaces (not multiple of 4)
        super().__init__(name, email, age)
        self.discount_rate = discount_rate

    def display(self) -> None:
        print(f"[VIP] {self.name} | {self.email} | 할인율: {self.discount_rate:.0%}")

    def get_discount(self) -> float:
        return self.discount_rate
```

**수정된 코드**:
```python
class VIPCustomer(Customer):
    def __init__(self, name: str, email: str, age: int,
                 discount_rate: float = 0.1):  # ✅ 17 spaces (aligned with opening paren)
        super().__init__(name, email, age)  # 부모 초기화 호출
        self.discount_rate = discount_rate  # VIP만의 속성

    def display(self) -> None:  # 오버라이딩
        print(f"[VIP] {self.name} | {self.email} | 할인율: {self.discount_rate:.0%}")

    def get_discount(self) -> float:
        return self.discount_rate
```

**설명**: 이 코드는 실제로 **실행 가능**합니다. Python은 들여쓰기 규칙이 유연하여 17개 공백도 문법적으로 유효합니다. 그러나 PEP 8 스타일 가이드에서는 들여쓰기를 4의 배수로 권장합니다. 현재 코드는 함수 매개변수를 여는 괄호에 정렬하는 스타일(hanging indent)을 사용하고 있으며, 이는 PEP 8에서 허용하는 방식입니다.

**권장 사항**: 코드는 동작하지만, 일관성을 위해 다음 중 하나를 선택하세요:
1. **Hanging indent 유지** (현재): 17개 공백 유지 (여는 괄호 정렬)
2. **4-space indent 사용**: 20개 공백으로 변경 (4의 배수)

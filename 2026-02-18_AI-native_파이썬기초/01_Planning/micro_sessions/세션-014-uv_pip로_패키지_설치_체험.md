## 🗣️ 강사 대본 구성

### 예상 분량
- **최소**: 공백 포함 1,500자
- **최대**: 공백 포함 2,000자
- **화법**: 친근한 구어체 (~해요, ~입니다)

### 서술 흐름 (반드시 포함)
1. **도입 (Hook)**: 텅 빈 냉장고를 채울 시간임을 선언 (2~3문장)
2. **비유 (Analogy)**: 밀키트를 인터넷에서 주문하는 과정 (5~8문장)
3. **개념 설명 (What)**: 파이썬 패키지를 설치하는 명령어의 구조(`uv pip install 패키지명`) (8~12문장)
4. **실제 활용 (How)**: 유명한 인터넷 요청 패키지인 `requests`를 설치해보고 속도 체감하기
5. **정리 (Checkpoint)**: 내 프로젝트가 인터넷의 자원을 끌어다 쓸 수 있게 되었음을 요약 (2~3문장)

### 예시 스크립트 (필수 - 작성자 참고용)
> "냉장고 문 열려있죠? `(.venv)` 표시가 보인다면 준비 완료입니다. 이제 쿠팡 로켓배송 매니저인 'uv'에게 주문을 넣어봅시다. 인터넷에 있는 자료를 가져오게 해주는 아주 유명한 밀키트가 있는데, 이름이 'requests'예요. 터미널에 `uv pip install requests`라고 치고 엔터를 눌러보세요. 눈 깜짝할 사이에 설치가 끝나버릴 겁니다. 예전 매니저(pip)가 자전거로 배달했다면, 우리 uv 매니저는 순간이동을 한 셈이죠. 이제 여러분의 빈 도구함에 외부 인터넷과 통신할 수 있는 강력한 기능이 추가된 겁니다."

## 🎙️ 실습 가이드 (해당 시)

### 실습 목표
- 이 실습을 통해 수강생은 외부 도구를 내 프로젝트 환경으로 가져오는 경험을 한다.

### 단계별 지시
| 단계 | 소요 시간 | 강사 지시사항 | 예상 결과 |
|------|----------|--------------|----------|
| 1 | 2분 | "터미널에 `(.venv)`가 있는지 다시 한번 확인하세요." | 가상환경 활성화 확인 |
| 2 | 5분 | "터미널에 `uv pip install requests`를 입력하세요." | 1초 내외로 설치 성공 메시지 출력 |
| 3 | 5분 | "`uv pip list`를 입력해 보세요." | 방금 설치한 requests 외 관련 도구들이 목록에 뜸 |
| 4 | 5분 | "AI 비서에게 'requests 패키지가 뭐야?'라고 물어보세요." | 해당 패키지에 대한 친절한 설명 획득 |

### 주의사항 및 트러블슈팅
- 흔한 오류 1: `(.venv)`가 없는 상태에서 설치함 (해결법: 컴퓨터 전체에 깔린 것이므로 치명적이지는 않으나, 다시 activate 후 재설치하여 개념을 명확히 함)
- 흔한 오류 2: 오타로 인한 설치 실패 (`request` 등) (해결법: 정확한 패키지명 입력의 중요성 설명)

## 💡 비유 및 스토리텔링

### 핵심 비유
- **비유**: 밀키트 주문과 냉장고 채우기
- **설명**: "requests 주세요!"라고 매니저(uv)에게 말하면, 매니저가 창고(인터넷)에서 정확한 밀키트를 찾아 내 전용 냉장고(.venv)에 채워 넣습니다.
- **AI 시대의 서사 연결**: 모든 도구를 직접 만들지 않고, 훌륭한 레고 블록을 가져다 조립하는 것이 현대 개발자의 기본 소양입니다.

### 스토리텔링 포인트
- 감정적 연결: "어? 너무 빨리 끝나서 뭔가 잘못된 거 아니에요?"라는 반응이 나올 정도로 uv의 속도가 빠름을 유쾌하게 언급.
- 전환 멘트: "자, 도구함도 완벽하고, 도구도 채워 넣었습니다. 드디어 길고 길었던 준비가 끝났어요. 이제 AI가 짜준 첫 번째 파이썬 코드를 실행해볼 역사적인 순간입니다."
